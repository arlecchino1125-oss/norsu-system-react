TECHNICAL ASPECTS OF THE NORSU – GUIHULNGAN CITY CAMPUS – LIBRARY’S ACTIVITY MANAGEMENT AND STUDENT MONITORING SYSTEM

SYSTEM ANALYSIS
The existing library management system at Negros Oriental State University – Guihulngan City Campus is primarily manual and lacks an integrated platform for efficiently managing student activities and library services. The current process requires students to manually log in and log out using record sheets, while library personnel monitor transactions and facility usage through paper-based records and spreadsheets. This method results in delays, inaccuracies, and difficulty in retrieving and organizing data.
Based on the data gathered, the current system shows several limitations. The manual recording of student attendance makes it difficult to accurately track the number of users and their duration of stay. Monitoring the utilization of library facilities, such as reading areas and computer laboratories, is also inefficient due to the absence of a centralized system. Additionally, book borrowing and returning transactions are not systematically recorded, leading to possible errors and inconsistencies in records.
The analysis further reveals that the existing system is time-consuming and prone to human error. Library staff experience difficulty in generating reports, as data must be manually compiled and verified. This affects the overall efficiency of library operations and limits the ability to make data-driven decisions. Furthermore, the lack of automation results in limited security and reliability of records.
To address these issues, the proposed Library’s Activity Management and Student Monitoring System introduces a centralized and automated platform that improves the overall functionality of library operations. The system allows students to log their activities digitally, enabling accurate and real-time tracking of library usage. It also provides a structured way to manage book transactions and monitor facility utilization.
The proposed system enhances data accuracy, reduces processing time, and minimizes errors associated with manual recording. It also enables faster retrieval of information and simplifies report generation for library personnel. With improved accessibility, usability, and security, the system supports better management of library resources and services.
Overall, the system analysis highlights the need for transitioning from a manual to an automated system. The proposed solution addresses the identified weaknesses of the current system and provides a more efficient, reliable, and user-friendly approach to library management.
SYSTEM DESIGN
The proposed Library’s Activity Management and Student Monitoring System is designed to provide an efficient, accurate, and centralized platform for managing library operations at Negros Oriental State University – Guihulngan City Campus. The system aims to replace the existing manual processes by introducing an automated solution that improves the monitoring of student activities, management of book transactions, and tracking of facility utilization.
The system is composed of two primary users: the administrator and the student. The administrator is responsible for managing the overall system, including monitoring student records, viewing activity logs, and handling book transactions. The student, on the other hand, interacts with the system by logging in using their credentials, recording their library visits, and requesting to borrow books.
The system follows a structured process wherein students input their required information, such as email, student number, name, and course, to access the system. Once logged in, their activities are automatically recorded and stored in the database. The system ensures that all data entries are organized and can be easily retrieved when needed. For book transactions, students can submit requests, which are then managed and monitored by the administrator.
The design of the system incorporates a centralized database that securely stores all records related to student activities and transactions. This ensures data consistency, reduces redundancy, and enhances the reliability of information. The system also includes features for generating reports, allowing administrators to easily monitor library usage and evaluate service performance.
In terms of interface, the system is designed to be user-friendly and accessible, ensuring ease of use for both students and administrators. Navigation is simplified to allow users to perform tasks efficiently without requiring extensive technical knowledge.
Overall, the system design focuses on improving operational efficiency, ensuring data accuracy, and enhancing the overall user experience. By integrating automation and centralized data management, the proposed system provides a reliable and effective solution to the challenges identified in the current library management process.

FIGURE 1FIGURE 1
This ERD represents a Library Services System with four main entities: User, Services, Book, and Borrow.
The User entity stores the information of people using the system. It has the attributes id as the primary key, email, and pass. The id uniquely identifies each user. This table is important because it connects users to the books they manage and the services they use.
The Services entity contains records related to service usage. Its attributes are id as the primary key, time_in, time_out, id_user as a foreign key, and opting. The id_user attribute links each service record to a specific user. This means one user can have multiple service records, showing a one-to-many relationship between User and Services.
The Book entity stores book-related data. It includes id as the primary key, book_title, and id_user as a foreign key. The id_user connects the book to a user, which means a user can be associated with multiple books. This also shows a one-to-many relationship between User and Book.
The Borrow entity serves as a transaction table that records borrowing activities. It contains id as the primary key, id_user, id_service, and id_book, all marked as foreign keys. These foreign keys connect the borrowing transaction to a specific user, service, and book. This table helps track which user borrowed which book under a certain service.
